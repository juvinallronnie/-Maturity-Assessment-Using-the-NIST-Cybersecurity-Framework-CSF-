GRC Mastery Capstone Project: Maturity Assessment Using the NIST Cybersecurity Framework (CSF)

Description of the Project
In this project, I acted as a junior GRC analyst to conduct a cybersecurity maturity assessment of a hypothetical organization using the NIST Cybersecurity Framework. I conducted a structured maturity assessment by analyzing the organization’s current cybersecurity environment. The assessment was based on responses to a set of predefined questions designed to capture key aspects such as team roles, existing controls, and operational processes. These responses were used to establish a baseline understanding of the organization’s current security posture. I then evaluated this information against the NIST CSF to identify gaps across the five core functions: Identify, Protect, Detect, Respond, and Recover. Based on the results of the assessment, I developed recommendations to address identified weaknesses and improve the organization’s overall cybersecurity maturity. This project applied the core principles of the NIST CSF—including its functions, categories, and gap analysis approach—to guide the assessment, rather than representing a full framework implementation.

Purpose of the Project
The purpose of this project was to gain practical experience applying GRC principles in a realistic scenario. It allowed me to understand how organizations assess their cybersecurity maturity and identify areas for improvement using a structured framework. The project also helped me develop the ability to analyze security controls, recognize gaps, and provide meaningful recommendations that align with industry best practices.

Tools Used
I used an Excel-based NIST CSF assessment tool to perform the evaluation. The tool was organized by the five core functions—Identify, Protect, Detect, Respond, and Recover—and included worksheets containing control categories, control objectives, and detailed assessment questions. I used this tool to document responses, track pass/fail results, identify control gaps, and add comments to support analysis and remediation planning. This approach reflects common industry practices, where Excel or GRC platforms are used to manage and track assessment data.

What I Learned
Through this project, I gained hands-on experience conducting a cybersecurity maturity assessment using the NIST CSF. I learned how to evaluate an organization’s current security posture by analyzing responses to structured assessment questions and mapping them to framework categories. I also developed the ability to identify control gaps and provide recommendations to improve cybersecurity practices. Additionally, I gained a better understanding of how GRC analysts use frameworks like NIST CSF to guide assessments and support risk-informed decision-making. This project strengthened my ability to think critically about cybersecurity and communicate findings in a clear, structured manner.
